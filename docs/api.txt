XÁC THỰC (AUTHENTICATION)
════════════════════════════════════════════════════════════════════════════════

Hệ thống bảo vệ API bằng JWT Token và phân quyền vai trò (RBAC).
Việc xác thực được thực hiện thông qua gRPC call tới NestJS Backend.

ROLE LEVELS:
  - 🔒 admin   : Toàn quyền truy cập, quản lý Metadata, Tệp tin và FAQ.
                 Thấy tất cả tệp tin và FAQ (kể cả lecturerOnly=true).
  - 🔒 lecture : Tương đương admin về quyền xem tài liệu và FAQ.
                 Thấy tất cả tệp tin và FAQ (kể cả lecturerOnly=true).
  - 🔒 auth    : Sinh viên đã xác thực (role=student).
                 Chỉ thấy tệp tin và FAQ có lecturerOnly=false.
                 Nếu JWT có enrollmentYear: chỉ thấy file không có metadata
                 năm học HOẶC file có enrollmentYearFrom ≤ year ≤ enrollmentYearTo.

GHI CHÚ: `lecturerOnly` là 1 trong 3 key metadata chuẩn áp dụng đồng nhất cho cả
tệp tin lẫn FAQ (cùng với enrollmentYear, academicYear) — dùng để pre-filter
tài liệu/FAQ trước khi đưa vào RAG traversal (xem Mục 5).

HEADER BẮT BUỘC:
  Authorization: Bearer <jwt_token>

JWT PAYLOAD (student):
  { "sub": 5, "role": "student", "enrollmentYear": 2022, ... }

LỖI THƯỜNG GẶP:
  - 401 Unauthorized: Thiếu token hoặc token hết hạn.
  - 403 Forbidden: Token hợp lệ nhưng Role không có quyền thực thi.

│      "type": "query_analysis",                                               │
│      "content": "Phân tích câu hỏi: ..."                                     │
│    }                                                                         │
│                                                                              │
│ 2. CORPUS_TRAVERSAL — Một hành động duyệt cây Corpus của agent              │
│    list roots, expand, inspect, select hoặc no-match; mỗi tool thành công   │
│    sinh một step realtime.                                                    │
│                                                                              │
│ 3. FAQ_RETRIEVAL / FAQ_ANSWER — FAQ được chọn và kết quả đủ/chưa đủ để trả lời.│
│    Hai step này chỉ xuất hiện khi Corpus traversal có FAQ candidate.         │
│                                                                              │
│ 4. FILE_RETRIEVAL — Tài liệu được hydrate/rerank sau khi FAQ chưa đủ.       │
│                                                                              │
│ 5. DOCUMENT_READ — Một PageIndex tool call đọc cấu trúc hoặc nội dung file. │
│                                                                              │
│ 6. TOOL_OUTPUT — Kết quả trả về thô của tool (Không lưu, không stream)      │
│    Lưu ý: Bị loại bỏ hoàn toàn khỏi cả stream lẫn DB để tiết kiệm băng thông │
│                                                                              │
│ 7. REASONING — Suy luận tổng hợp của Agent (chỉ stream, không lưu DB)       │
│    Nguồn: (a) part.thought nội bộ từ Gemini API, hoặc                        │
│           (b) Status text từ service.py khi gọi tool (tiếng Việt thân thiện) │
│           (c) Văn bản Agent tự viết trước thẻ <answer> theo system prompt    │
│    Ví dụ: {"type": "reasoning",                                              │
│            "content": "Đang tra cứu mục lục của 'Quy chế đào tạo ĐH 2021'...",│
│            "done": false}                                                    │
│                                                                              │
│ 8. TEXT — Nội dung câu trả lời cuối cùng (chỉ trong /stream)                │
│    {"type": "text", "content": "Học phí CLC năm 2025 là...", "done": false} │
│                                                                              │
│ THỨ TỰ ĐIỂN HÌNH CỦA CÁC EVENT TRONG MỘT /stream REQUEST:                  │
│   query_analysis → corpus_traversal × N → faq_retrieval → faq_answer        │
│   → text nếu FAQ đủ, hoặc file_retrieval → document_read × N                │
│   → reasoning(kết luận) → text(s) → done                                    │
│                                                                              │
│ LƯU VÀO MONGODB:                                                             │
│   query_analysis, corpus_traversal, faq_retrieval, faq_answer,              │
│   file_retrieval, document_read. Không lưu tool_output, reasoning, text.     │
└──────────────────────────────────────────────────────────────────────────────┘

                        10. APPENDIX: ERROR CODES
════════════════════════════════════════════════════════════════════════════════

- 400 Bad Request: Lỗi tham số (ValidationException).
- 401 Unauthorized: JWT token không hợp lệ (gRPC Auth reject).
- 403 Forbidden: Sai Role người dùng (ForbiddenException).
- 404 Not Found: Dữ liệu không tồn tại trong DB.
- 409 Conflict: Dữ liệu đã tồn tại hoặc đang bị ràng buộc (Metadata in use).
- 500 Internal Error: Lỗi hạ tầng (MongoDB, R2).
- 429 Too Many Requests: Lỗi Rate Limit từ Gemini API (Tự động retry ở client).

--------------------------------------------------------------------------------
LƯU Ý VỀ LƯU TRỮ VÀ CACHING (STORAGE & CACHE):
- Tài liệu Markdown (.md) được cache tại local server (`PAGEINDEX_WORKSPACE/artifacts`).
- Thời gian tồn tại (TTL): 60 phút kể từ lần truy cập cuối.
- Background Task: Hệ thống tự động quét và dọn dẹp các tệp quá hạn mỗi 30 phút.
- R2 Fallback: Nếu cache bị dọn dẹp, hệ thống sẽ tự động tải lại từ R2 khi có yêu cầu.
- Agent Control: Số vòng lặp tối đa của RAG Agent được cấu hình qua `AGENT_MAX_TURNS` (mặc định: 7).
- Performance Logging: Hệ thống ghi chi tiết thời gian xử lý (Ingestion phases, Retrieval, Agent/Tool duration) vào logs phục vụ tối ưu hóa.

LƯU Ý VỀ RETRIEVAL (CORPUS TREE ARCHITECTURE):
- File retrieval dùng Corpus Traversal: Duyệt cây chủ đề từ gốc, gọi LLM phân tích và đi xuống các nhánh con cho đến khi tìm thấy tài liệu liên quan.
  Không dùng vector embedding hay Qdrant; file candidates được rerank bằng Cohere nếu bật config.
- FAQ answering: LLM đọc nội dung FAQ liên quan và trả answer/null theo mức độ đủ bao phủ câu hỏi.
  Không dùng vector embedding hay Qdrant; FAQ docs cũng được rerank riêng bằng Cohere. Tuning qua COHERE_RERANK_* và FAQ_MATCHER_MAX_CATALOG.
