XÁC THỰC (AUTHENTICATION)
════════════════════════════════════════════════════════════════════════════════

Hệ thống bảo vệ API bằng JWT Token và phân quyền vai trò (RBAC). 
Việc xác thực được thực hiện thông qua gRPC call tới NestJS Backend.

ROLE LEVELS:
  - 🔒 admin: Toàn quyền truy cập, quản lý Metadata và Tệp tin.
  - 🔒 auth: Mọi người dùng đã xác thực (Student/Lecture) đều có quyền xem toàn bộ tài liệu công khai và metadata tags. Phân quyền chỉ áp dụng ở tầng bảo mật Auth chung, không lọc theo file scope.

HEADER BẮT BUỘC:
  Authorization: Bearer <jwt_token>

LỖI THƯỜNG GẶP:
  - 401 Unauthorized: Thiếu token hoặc token hết hạn.
  - 403 Forbidden: Token hợp lệ nhưng Role không có quyền thực thi.

                        10. APPENDIX: ERROR CODES
════════════════════════════════════════════════════════════════════════════════

- 400 Bad Request: Lỗi tham số (ValidationException).
- 401 Unauthorized: JWT token không hợp lệ (gRPC Auth reject).
- 403 Forbidden: Sai Role người dùng (ForbiddenException).
- 404 Not Found: Dữ liệu không tồn tại trong DB.
- 409 Conflict: Dữ liệu đã tồn tại hoặc đang bị ràng buộc (Metadata in use).
- 500 Internal Error: Lỗi hạ tầng (Qdrant, MongoDB, R2, Gemini API).

LƯU Ý VỀ LƯU TRỮ VÀ CACHING (STORAGE & CACHE):
- Tài liệu Markdown (.md) được cache tại local server (`PAGEINDEX_WORKSPACE/artifacts`).
- Thời gian tồn tại (TTL): 60 phút kể từ lần truy cập cuối.
- Background Task: Hệ thống tự động quét và dọn dẹp các tệp quá hạn mỗi 30 phút.
- R2 Fallback: Nếu cache bị dọn dẹp, hệ thống sẽ tự động tải lại từ R2 khi có yêu cầu.
- Agent Control: Số vòng lặp tối đa của RAG Agent được cấu hình qua `AGENT_MAX_TURNS` (mặc định: 7).
- Performance Logging: Hệ thống ghi chi tiết thời gian xử lý (Ingestion phases, Retrieval, Agent/Tool duration) vào logs phục vụ tối ưu hóa.

Hoàn tất bản Đặc tả API chi tiết v5.3.0 (Full Compliance).
